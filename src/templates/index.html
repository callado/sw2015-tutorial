Explanation for the different results:

As seen above, the results for the same query were different depending whether or not the reasoner was used. In the first query, the one that did not use the reasoner, we get as a result only the instances that were asserted as "Food" - that is, only the ones that had been defined manually as a member of the class "Food" on the database. When we activate the reasoner, though, we get much more results. This happens because, in this ontology, the class "Food" is defined as anything that has the property "providesNutrients". Therefore, as every instance of the class "Ingredient" has a property "providesNutrients", the reasoner infers that it is also a member of the class "Food".
